## Usage

### Viewing All Templates

```bash
aifiles-templates list
```

### Creating Folder Structure

```bash
# Create folders for a specific template
aifiles-templates create-folders documents

# Create folders for all templates
aifiles-templates create-folders downloads
aifiles-templates create-folders work-documents
aifiles-templates create-folders development
```

### Enabling Auto-Organization

```bash
# Enable auto-organization for a template
aifiles-templates enable downloads

# Enable watch mode (requires auto-organization)
aifiles-templates enable pictures
```

### Customizing Templates

```bash
# Modify a template
aifiles-templates edit documents

# Add custom folder structure
aifiles-templates import-structure documents ./my-structure.txt

# Export template to share
aifiles-templates export documents ./documents-template.json
```

## Template Selection by AI

When using multi-call analysis, AIFiles automatically selects the best template based on file content:

```bash
aifiles file.pdf
# AI analyzes file and selects appropriate template
# e.g., "work-documents" for business reports
```

The AI considers:
- File content and purpose
- Category and subcategories
- Detected metadata
- Naming structure compatibility
- Base path alignment

## Naming Conventions

Templates use consistent naming patterns:

- **kebab-case**: `my-file-name` (most templates)
- **snake_case**: `my_file_name` (some legacy compatibility)
- **camelCase**: `myFileName` (desktop template)

## Auto-Organization

**Enabled by default:**
- Downloads (most active)
- Desktop (high activity)
- Pictures (frequent additions)
- Videos (regular updates)
- Music (library updates)

**Disabled by default:**
- All specialized templates (work, business, development, etc.)
- Reason: Users prefer manual control over important documents

## Watch Mode

**Enabled by default:**
- Downloads (automatic cleanup)
- Desktop (real-time organization)
- Pictures (photo management)

**Disabled by default:**
- All other templates
- Reason: Prevents unexpected file movements

## Best Practices

### 1. Start with Core Templates
Enable only what you need:
```bash
aifiles-templates enable downloads
aifiles-templates enable desktop
aifiles-templates enable pictures
```

### 2. Create Folder Structures Early
Set up structures before adding files:
```bash
aifiles-templates create-folders work-documents
aifiles-templates create-folders development
```

### 3. Customize for Your Workflow
Modify templates to match your needs:
```bash
# Add your own folder structure
aifiles-templates import-structure work-documents ./my-folders.txt
```

### 4. Use Watch Mode Strategically
Enable watching only for high-activity folders:
```bash
aifiles watch ~/Downloads
aifiles watch ~/Desktop
```

### 5. Export for Backup
Save your customized templates:
```bash
aifiles-templates export work-documents ./backup.json
```

## Migration from Old Config

If you have an existing AIFiles installation, these templates will be created on first run. Your existing templates will be preserved and merged with defaults.

## Customization Guide

Each template can be fully customized:

1. **Folder Structure**: Add/remove folders as needed
2. **Naming Pattern**: Change `{file_category_1}/{file_title}` format
3. **Base Path**: Modify `~/Documents` to any location
4. **File Name Case**: Choose kebab, snake, camel, pascal, etc.
5. **Auto-Organization**: Enable/disable automatic processing
6. **Watch Mode**: Enable/disable real-time monitoring

See [FOLDER-TEMPLATES-README.md](../examples/FOLDER-TEMPLATES-README.md) for detailed customization instructions.
